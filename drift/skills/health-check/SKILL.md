## 目标
检查 MEMORY.md 和 HISTORY.md 中是否存在矛盾或过时条目，确保用户画像的准确性。

## 步骤
1. 读取 MEMORY.md 全文
2. 读取 HISTORY.md 最近 30 天的条目
3. 逐条交叉比对：
   - HISTORY 中记录的事件是否和 MEMORY 中的用户画像矛盾？
   - MEMORY 中是否有明显过时的信息（如已毕业但写"在读"）？
4. 发现矛盾 → 向用户确认后修正
5. 发现过时信息 → 标记为待更新
6. 输出审计报告（矛盾数量、过时数量、建议操作）
